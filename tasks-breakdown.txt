1. Api Endpoints (/signup, /login, /validate, /buy, and /sell)
2. 500,000 add money to user wallet on signup
3. Buy Endpoint
  a. current price of bitcoin 
  b. amount of bitcoin user wants to purchase
  c. validate for enough money & proceed to increment bitcoin value
  d. decrement wallet balance to represent new-balance
  e. return current bitcoin & wallet details
4. Sell Endpoint
  a. current price of bitcoin 
  b. validate for enough bitcoin to see & decrease bitcoin value
  c. increment wallet amount 
  d. return current bitcoin & wallet details 


purchase flow:
  params:
    1. get Current Price of BitCoin (ideally should be check from BE)
    2. Amount of BitCoin user wants to purchase

  validations:
    1. check if amount is their in users wallet balance

  success:
    1. increment bitcoin balance & amount
    2. decrement users wallet balance

  response: 
    1. return updated wallet & bitcoin details
